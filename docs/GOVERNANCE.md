# Governance

This document describes the governance model for the Aurora LTFS project.

## Roles

### Core Member

Core Members are the maintainers of the project. They are responsible for the direction, quality, and releases of Aurora LTFS.

**Permissions:**

- Direct commit access to all branches except release branches.
- Merge pull requests after review.
- Manage issues, labels, and milestones.

**Responsibilities:**

- Review and approve pull requests. All external pull requests require approval from at least one Core Member before merging.
- Maintain code quality and adherence to project standards.
- Triage issues and guide contributors.

### External Contributors

Anyone who is not a Core Member contributes via pull requests.

**Process:**

- Fork the repository and submit a pull request.
- All pull requests are reviewed by Core Members.
- At least one Core Member approval is required before merging.

## Becoming a Core Member

Core Members may invite new members based on sustained, high-quality contributions and demonstrated understanding of the project. There is no formal application process at this time.

## Changes to Governance

This governance model may evolve as the project and community grow. Changes will be discussed openly via GitHub Issues.
